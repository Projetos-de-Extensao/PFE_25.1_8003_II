Introdução

O Brainstorm é uma técnica usada para levantar requisitos, que envolve reunir a equipe para discutir vários aspectos gerais do projeto apresentados no documento do problema de negócios. Durante o brainstorm, o diálogo é estimulado e as críticas são evitadas, permitindo que todos contribuam com suas próprias ideias.

Metodologia

A equipe se reuniu para debater ideias gerais sobre um projeto de um site de eventos, começou com muitas ideias e terminou com um objetivo traçado com o nome do site sendo definido de Festify onde o John Carvalho foi o moderador, direcionando a equipe com questões pré-elaboradas, e transcrevendo as respostas para o documento.

Versão 1.0

Perguntas

1. Qual o objetivo principal da aplicação?

- Deve ser uma plataforma onde qualquer pessoa possa comprar ingressos para diversos eventos, tendo o foco em festas e shows.

- A plataforma deve fornecer a venda de ingressos para shows e festas em todo o Brasil.

- O objetivo da aplicação é proporcionar a melhor experiência ao usuário.

- O principal objetivo da aplicação é a maior simplicidade possível para o usuário procurar e comprar o que deseja.

- A plataforma deve gerenciar diversos aspectos relacionados à organização e promoção de eventos.

2. Como será o processo para cadastrar um novo cliente?

- O moderador deverá acessar o sistema e fazer login.

- O cliente (organizador de eventos) deverá preencher um formulário com seus dados comerciais, como nome, e-mail, telefone e redes sociais. - Com o usuário logado, ele deverá confirmar os dados e enviar para validação. - O cliente receberá uma notificação quando seu cadastro for aprovado. - O cliente, uma vez aprovado, poderá acessar o painel para cadastrar eventos e gerenciar ingressos.

--- ### 3. Como será a forma de adicionar produtos?

- O cliente, ao cadastrar um evento, poderá adicionar produtos como ingressos, pacotes VIP ou brindes exclusivos.

- O produto tem informações como nome, descrição, preço e quantidade disponível.

- O produto pode ter variações, como ingressos de diferentes setores ou benefícios extras. - produto será listado na página do evento para compra pelos usuários.

--- ### 4. Outras perguntas pertinentes ao contexto

- Com a localização ativada, o site pode sugerir eventos próximos ao usuário. - O cliente poderá visualizar estatísticas de vendas de ingressos e engajamento do público. - O cliente poderá criar promoções ou cupons de desconto para incentivar vendas. --- ### 5. "Outras perguntas pertinentes ao contexto", Como seria a forma de adicionar do cliente adicionar os produtos ?

- O cliente acessa o painel e seleciona um evento já cadastrado.

- No evento, ele pode adicionar ingressos, brindes ou serviços extras. - Ele define nome, preço, disponibilidade e descrição do produto. - O sistema exibe os produtos na página do evento para os usuários comprarem.

### 6. Quais informações seriam interessante para o cliente?

- Informações sobre a performance dos eventos (vendas, visualizações, engajamento). - O cliente usuário poderá acessar informações sobre tendências do mercado de eventos e sugestões de melhorias. - O usuário poderá visualizar estatísticas de vendas, acessos à página do evento e feedback dos participantes. - O usuário poderá ver relatórios sobre a quantidade de ingressos vendidos, ingressos restantes e horários de pico de vendas.
